## 这是什么？

Gato Neura是DeepMind的AlphaZero核心思想的**自包含教育实现**，规模缩小到井字棋，以便完全在Web浏览器中运行。

它展示了：
- **双头神经网络**（策略 + 价值）
- **自我对弈强化学习**
- **蒙特卡洛树搜索（MCTS）** 用于走法选择
- **课程学习**（随机对手 → 完美极小极大 → 自身快照）
- **实时可视化**权重、架构和训练指标
- **模型保存/加载** — 将训练好的模型下载为JSON并分享

全部代码仅约2,000行vanilla JavaScript，**零构建步骤**。打开DevTools即可阅读实际运行的代码。

## 如何使用

界面分为三个面板：

### 1. 模型面板（左侧）
显示神经网络架构的实时SVG图。点击任意节点查看其传入权重的热力图。

### 2. 棋盘面板（中间）
游戏本身。选择谁先手，设置游戏温度，与训练好的网络对弈。

- **游戏温度**：`0.0` = 总是最佳走法（贪心）。`>1.0` = 有创意，有时随机。
- **策略提示**：在网络的回合，每个空格会显示百分比，表示网络的置信度。

### 3. 控制面板（右侧）

#### 模型架构
训练前添加/移除隐藏层。默认：`[64, 32]`。

> **提示：** 对于井字棋，2-3层32-64个神经元已经足够。更多层不会帮助，反而可能减缓收敛。

#### 训练设置

| 设置 | 默认值 | 作用 |
|------|--------|------|
| **对局数** | 500 | 生成的自我对弈总局数 |
| **学习率** | 0.0003 | 梯度下降的步长 |
| **批量大小** | 64 | 每次更新的训练样本数 |
| **MCTS模拟次数** | 50 | 每步模拟次数（0 = 禁用） |
| **增量训练** | 关 | 添加到现有模型或从头开始 |

#### 训练按钮
点击**训练**，实时观察指标更新。训练期间游戏界面会暂停。

## 理解指标

### 胜率
网络在当前训练会话中获胜的对局百分比。

| 徽章 | 含义 |
|------|------|
| 🟢 优秀 (>80%) | 完全压制当前对手 |
| 🔵 良好 (60-80%) | 赢得大部分，仍在学习 |
| 🟡 一般 (40-60%) | 尚未压制 |
| 🔴 较差 (<40%) | 输多赢少 |

> 训练胜率针对**混合对手**（随机 + 旧快照）。真正的测试是**对战随机质量**。

### 策略损失
网络预测*走哪一步*的能力。

越低越好。9个格子时，纯随机猜测的基准损失为：

$$\mathcal{L}_{\text{random}} = -\log\frac{1}{9} = 2.197$$

损失2.1意味着只比掷骰子好约5%。

| 徽章 | 阈值 |
|------|------|
| 🟢 优秀 | < 1.0 |
| 🔵 良好 | 1.0 - 1.5 |
| 🟡 一般 | 1.5 - 2.0 |
| 🔴 较差 | > 2.0 |

### 价值损失
网络评估局面是赢(+1)、输(-1)还是平局(0)的能力。

这是均方误差(MSE)：

$$\mathcal{L}_V = \frac{1}{N}\sum_{i=1}^{N}(v_i^{\text{target}} - v_i^{\text{pred}})^2$$

| 徽章 | 阈值 |
|------|------|
| 🟢 优秀 | < 0.3 |
| 🔵 良好 | 0.3 - 0.6 |
| 🟡 一般 | 0.6 - 1.0 |
| 🔴 较差 | > 1.0 |

> 低价值损失不等于下得好。网络可能评估很好，但不知道*该走什么*。

### 对战随机质量
训练结束后，网络进行**100局贪心对局**对抗纯随机对手。这是 ground-truth 指标。

| 结果 | 解读 |
|------|------|
| >90% | 接近完美。击败任何 casual 玩家。 |
| 70-90% | 很强，偶尔犯错。 |
| 50-70% | 懂基础，漏掉战术。 |
| <50% | 比随机还差。需要更多训练或更大的网络。 |

实时绘制最近100个训练批次的策略损失（蓝色）和价值损失（红色）。

## 背后的数学

### 神经网络架构

网络是一个具有两个输出头的**多层感知机**：

$$
\mathbf{x} \in \mathbb{R}^9 \xrightarrow{\text{全连接}} \mathbf{h}_1 \xrightarrow{\text{ReLU}} \mathbf{h}_2 \xrightarrow{\text{ReLU}} \begin{cases} \mathbf{p} \in \mathbb{R}^9 & \text{（策略）} \\ v \in \mathbb{R} & \text{（价值）} \end{cases}
$$

**输入编码：**
- $+1$：网络的棋子
- $-1$：对手的棋子
- $0$：空格

**视角不变性：** 网络总是将自己视为 $+1$。如果作为O下棋，输入前会翻转棋盘。

### 策略头

使用softmax输出9个格子的概率分布：

$$p_i = \frac{e^{z_i}}{\sum_{j} e^{z_j}}$$

非法走法（已占格子）的概率被掩码为0，然后重新归一化。

### 价值头

单个神经元使用 $\tanh$ 激活，输出 $[-1, +1]$ 的标量：

$$v = \tanh(w^T h + b)$$

### 策略损失：交叉熵

网络被训练以匹配目标策略 $\pi$（来自MCTS访问计数）：

$$\mathcal{L}_P = -\frac{1}{N}\sum_{i=1}^{N}\sum_{j=1}^{9} \pi_{ij} \log \hat{p}_{ij}$$

为数值稳定性进行裁剪（$\varepsilon = 10^{-7}$）：

$$\hat{p}_{ij} = \text{clip}(p_{ij}, \varepsilon, 1-\varepsilon)$$

### 价值损失：均方误差

$$\mathcal{L}_V = \frac{1}{N}\sum_{i=1}^{N}(z_i - v_i)^2$$

其中 $z_i \in \{-1, 0, +1\}$ 是游戏结果。

### 总损失

$$\mathcal{L} = \mathcal{L}_P + \mathcal{L}_V$$

### 温度采样

训练期间，以温度 $T$ 采样走法来平衡探索与利用：

$$\tilde{p}_i = \frac{p_i^{1/T}}{\sum_j p_j^{1/T}}$$

- $T = 1.0$：从原始策略采样（高探索）
- $T \to 0$：总是选择最高概率（纯利用）
- 本实现中，$T$ 在训练期间从1.0线性衰减到0.3。

### 蒙特卡洛树搜索（MCTS）

约50局后激活MCTS。每步运行 $N$ 次模拟：

1. **选择**：使用UCB1分数遍历游戏树：

$$U(s, a) = Q(s, a) + c_{puct} \cdot P(a|s) \cdot \frac{\sqrt{N(s)}}{1 + N(s, a)}$$

2. **扩展**：到达未访问节点时，使用网络的策略输出扩展。
3. **评估**：对叶节点使用网络的价值估计。
4. **回传**：将结果向上传播，每层翻转符号（零和博弈）。
5. **走法选择**：按访问计数比例下棋。

每次模拟需要网络一次前向传播，所以50次模拟 ≈ 比贪心慢50倍。

### 梯度裁剪

为防止梯度爆炸，将梯度裁剪到最大范数：

$$\mathbf{g} \leftarrow \frac{\mathbf{g}}{\max(1, \|\mathbf{g}\|_2 / g_{\max})}$$

本实现中，$g_{\max} = 1.0$。

## 架构

```
public/
├── index.html              # 主页面
├── css/styles.css          # 响应式布局
└── js/
    ├── app.js              # 编排器，游戏循环
    ├── game.js             # 纯游戏逻辑（不可变）
    ├── model.js            # TF.js 模型创建/推理
    ├── trainer.js          # 自我对弈 + MCTS + 训练循环
    ├── ui.js               # DOM绑定，指标，i18n
    ├── visualizer.js       # SVG模型渲染
    ├── i18n.js             # 轻量级i18n引擎
    └── translations/       # en, es, zh, ja
        ├── en.js
        ├── es.js
        ├── zh.js
        └── ja.js
```

**设计理念：** 零构建步骤。DevTools中看到的代码就是实际运行的代码。无webpack，无npm，无转译。

## 训练阶段

训练课程分为三个阶段：

| 阶段 | 对局 | 对手 | MCTS | 目的 |
|------|------|------|------|------|
| 1 | 0-50 | 随机 | 否 | 学习基本获胜模式 |
| 2 | 50-100 | 随机 + 快照 | 是 | 通过树搜索学习战术 |
| 3 | 100+ | 快照（自身） | 是 | 针对过去版本优化 |

每100局对模型进行**快照**。这个旧版本成为对手，迫使当前模型对抗更强的敌人。
