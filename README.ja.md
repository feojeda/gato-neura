このプロジェクトは、TensorFlow.jsを使用した**自己対弈による強化学習**、**モンテカルロ木探索（MCTS）**、**デュアルヘッドニューラルネットワーク**（ポリシー + バリュー）を実演します。すべてがブラウザ内でローカルに実行されます。リアルタイムで学習を観察したり、重みを検査したり、意思決定を可視化したり、訓練済みモデルをダウンロードしたり、実際に実行されている正確なソースコードを読んだりできます。

## これは何？

Gato Neuraは、DeepMindのAlphaZeroの核心的アイデアを**自己完結型の教育実装**として、三目並べにスケールダウンし、Webブラウザ内で完全に動作するようにしたものです。

以下を実演します：
- **デュアルヘッドニューラルネットワーク**（ポリシー + バリュー）
- **自己対弈による強化学習**
- **モンテカルロ木探索（MCTS）** による着手選択
- **カリキュラム学習**（ランダム相手 → 完璧なミニマックス → 自身のスナップショット）
- **重み、アーキテクチャ、学習指標のリアルタイム可視化**
- **モデルの保存/読み込み** — 訓練済みモデルをJSONとしてダウンロードして共有

すべてがバニラJavaScript約2,000行で、**ビルドステップゼロ**。DevToolsを開けば、実際に動作しているコードをそのまま読めます。

## 使い方

UIは3つのパネルに分かれています：

### 1. モデルパネル（左）
ニューラルネットワークのアーキテクチャをリアルタイムのSVGグラフで表示。ノードをクリックすると、入力重みのヒートマップが表示されます。

### 2. ボードパネル（中央）
ゲーム本体。先手を選び、プレイ温度を設定し、学習済みネットワークと対戦します。

- **プレイ温度**：`0.0` = 常に最善手（greedy）。`>1.0` = クリエイティブで、時々ランダム。
- **ポリシーヒント**：ネットワークの番の間、各空きマスにパーセンテージが表示され、ネットワークの信頼度を示します。

### 3. コントロールパネル（右）

#### モデルアーキテクチャ
学習前に隠れ層を追加/削除。デフォルト：`[64, 32]`。

> **ヒント：** 三目並べでは、32-64ニューロンの層を2-3層あれば十分です。それ以上は逆に収束を遅らせる可能性があります。

#### 学習設定

| 設定 | デフォルト | 役割 |
|------|-----------|------|
| **対局数** | 500 | 生成する自己対弈の総数 |
| **学習率** | 0.0003 | 勾配降下のステップサイズ |
| **バッチサイズ** | 64 | 1回の更新あたりの学習サンプル数 |
| **MCTSシミュレーション数** | 50 | 1手あたりのシミュレーション数（0 = 無効） |
| **増分学習** | オフ | 既存モデルに追加するか、最初からやり直すか |

#### 学習ボタン
**学習**を押すと、指標がリアルタイムで更新されます。学習中はゲームUIが一時停止します。

## 指標の理解

### 勝率
現在の学習セッション中にネットワークが勝った対局の割合。

| バッジ | 意味 |
|--------|------|
| 🟢 優秀 (>80%) | 現在の相手を圧倒している |
| 🔵 良好 (60-80%) | 大半に勝っている、まだ学習中 |
| 🟡 普通 (40-60%) | まだ圧倒的でない |
| 🔴 不良 (<40%) | 勝つより負けている |

> 学習勝率は**混合相手**（ランダム + 古いスナップショット）に対するものです。真のテストは**ランダム対戦品質**です。

### ポリシー損失
ネットワークが*どの手を指すべきか*を予測する能力。

低いほど良い。9マスの場合、純粋なランダム推測のベースライン損失は：

$$\mathcal{L}_{\text{random}} = -\log\frac{1}{9} = 2.197$$

損失2.1は、サイコロよりわずか5%良いだけを意味します。

| バッジ | 閾値 |
|--------|------|
| 🟢 優秀 | < 1.0 |
| 🔵 良好 | 1.0 - 1.5 |
| 🟡 普通 | 1.5 - 2.0 |
| 🔴 不良 | > 2.0 |

### バリュー損失
局面が勝ち(+1)、負け(-1)、引き分け(0)のどれかを評価する能力。

平均二乗誤差(MSE)：

$$\mathcal{L}_V = \frac{1}{N}\sum_{i=1}^{N}(v_i^{\text{target}} - v_i^{\text{pred}})^2$$

| バッジ | 閾値 |
|--------|------|
| 🟢 優秀 | < 0.3 |
| 🔵 良好 | 0.3 - 0.6 |
| 🟡 普通 | 0.6 - 1.0 |
| 🔴 不良 | > 1.0 |

> 低いバリュー損失 ≠ うまくプレイすること。評価はできても、*何を指すべきか*わからない可能性があります。

### ランダム対戦品質
学習後、ネットワークは純粋にランダムな相手に対して**100局のgreedy対局**を行います。これが真の指標です。

| 結果 | 解釈 |
|------|------|
| >90% | ほぼ完璧。カジュアルな人間には負けない。 |
| 70-90% | 強いが、時々ミスをする。 |
| 50-70% | 基礎は知っているが、戦術を見逃す。 |
| <50% | ランダムより悪い。より多くの学習か、より大きなネットワークが必要。 |

### 損失グラフ
直近100バッチのポリシー損失（青）とバリュー損失（赤）をリアルタイムにプロットします。

## 背後にある数学

### ニューラルネットワークアーキテクチャ

ネットワークは2つの出力ヘッドを持つ**多層パーセプトロン**です：

$$
\mathbf{x} \in \mathbb{R}^9 \xrightarrow{\text{全結合}} \mathbf{h}_1 \xrightarrow{\text{ReLU}} \mathbf{h}_2 \xrightarrow{\text{ReLU}} \begin{cases} \mathbf{p} \in \mathbb{R}^9 & \text{（ポリシー）} \\ v \in \mathbb{R} & \text{（バリュー）} \end{cases}
$$

**入力エンコーディング：**
- $+1$：ネットワークの駒
- $-1$：相手の駒
- $0$：空き

**視点の不変性：** ネットワークは常に自分自身を $+1$ として認識します。Oとしてプレイする場合、入力前にボードが反転されます。

### ポリシーヘッド

softmaxを使用して9マスの確率分布を出力します：

$$p_i = \frac{e^{z_i}}{\sum_{j} e^{z_j}}$$

違法な手（占有済みマス）は確率0にマスクされ、分布が再正規化されます。

### バリューヘッド

単一のニューロンは $\tanh$ 活性化を使用し、$[-1, +1]$ のスカラーを出力します：

$$v = \tanh(w^T h + b)$$

### ポリシー損失：交差エントロピー

ネットワークは、MCTSの訪問カウントから得られる目標ポリシー $\pi$ と一致するように訓練されます：

$$\mathcal{L}_P = -\frac{1}{N}\sum_{i=1}^{N}\sum_{j=1}^{9} \pi_{ij} \log \hat{p}_{ij}$$

数値安定性のためにクリッピング（$\varepsilon = 10^{-7}$）：

$$\hat{p}_{ij} = \text{clip}(p_{ij}, \varepsilon, 1-\varepsilon)$$

### バリュー損失：平均二乗誤差

$$\mathcal{L}_V = \frac{1}{N}\sum_{i=1}^{N}(z_i - v_i)^2$$

ここで $z_i \in \{-1, 0, +1\}$ はゲーム結果です。

### 総損失

$$\mathcal{L} = \mathcal{L}_P + \mathcal{L}_V$$

### 温度サンプリング

学習中、温度 $T$ で手をサンプリングして探索と活用のバランスを取ります：

$$\tilde{p}_i = \frac{p_i^{1/T}}{\sum_j p_j^{1/T}}$$

- $T = 1.0$：生のポリシーからサンプリング（高い探索）
- $T \to 0$：常に最高確率を選択（純粋な活用）
- 本実装では、$T$ は学習中に1.0から0.3へ線形に減衰します。

### モンテカルロ木探索（MCTS）

約50局後にMCTSが有効化されます。各手で $N$ 回のシミュレーションを実行します：

1. **選択**：UCB1スコアを使ってゲーム木を走査します：

$$U(s, a) = Q(s, a) + c_{puct} \cdot P(a|s) \cdot \frac{\sqrt{N(s)}}{1 + N(s, a)}$$

2. **展開**：未訪問ノードに到達すると、ネットワークのポリシー出力を使って展開します。
3. **評価**：葉ノードにはネットワークのバリュー推定を使用します。
4. **バックアップ**：結果を上に伝播し、各レベルで符号を反転します（ゼロサムゲーム）。
5. **着手選択**：訪問カウントに比例して着手します。

各シミュレーションにはネットワークのフォワードパスが1回必要なので、50回のシミュレーションはgreedyより約50倍遅いです。

### 勾配クリッピング

勾配爆発を防ぐため、勾配を最大ノルムにクリッピングします：

$$\mathbf{g} \leftarrow \frac{\mathbf{g}}{\max(1, \|\mathbf{g}\|_2 / g_{\max})}$$

本実装では、$g_{\max} = 1.0$ です。

## 学習フェーズ

学習カリキュラムは3つのフェーズに分かれています：

| フェーズ | 対局 | 相手 | MCTS | 目的 |
|----------|------|------|------|------|
| 1 | 0-50 | ランダム | なし | 基本的な勝ちパターンを学習 |
| 2 | 50-100 | ランダム + スナップショット | あり | 木探索を通じて戦術を学習 |
| 3 | 100+ | スナップショット（自身） | あり | 過去の自身に対して洗練 |

モデルの**スナップショット**は100局ごとに撮影されます。この古いバージョンが相手となり、現在のモデルがより強い敵に対して改善するように強制されます。
